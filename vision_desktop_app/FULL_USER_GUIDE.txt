1. Введение
-----------
Данное приложение представляет собой рабочее окружение для аннотации изображений, обучения моделей на TensorFlow без использования заранее предобученных весов и тестового предсказания. Оно поставляется с простым GUI (PySide6), набором шаблонов моделей и набором инструментов для генерации тренировочного датасета и запуска короткого примера обучения.

------------------------------------------------

4. Аннотация изображений в приложении
-------------------------------------
1) Откройте вкладку "Аннотация" и нажмите "Загрузить изображение".
2) В поле "Название метки" введите текст (например, "человек", "коробка", "земля").
3) Нарисуйте bounding box на изображении: нажмите левой кнопкой мыши в начале области, проведите и отпустите.
4) Каждая рамка автоматически получает текущее название метки, но вы можете повторно ввести новое имя для следующей рамки.
5) Нажмите "Сохранить аннотацию" — в папке с изображением будет сохранён JSON-файл того же имени, но с расширением `.json`.

Формат JSON-аннотаций (напр., `image_0000.json`):
{
  "image": "image_0000.png",
  "annotations": [
    {"label": "person", "x": 10, "y": 20, "w": 50, "h": 60},
    {"label": "box", "x": 70, "y": 30, "w": 40, "h": 35}
  ]
}

Замечание: координаты x,y,w,h указаны в пикселях относительно оригинального изображения (не масштабированного).

5. Подготовка датасета и структура папок
----------------------------------------
- Структура данных для обучения должна выглядеть так:

```
data/my_dataset/
  image_0000.png
  image_0000.json
  image_0001.png
  image_0001.json
  ...
```

- JSON содержит список аннотаций (возможно несколько объектов на изображении). Parser приведёт к одному bbox на изображение в простом детекторе (пока что поддерживается single bbox).
- Для классификации укажите метки как отдельные имена в `.json` (parser берёт первую метку и преобразует её в индекс).

6. Обучение моделей (с учителем)
--------------------------------
6.1 Классификация
- Поведение: модель предсказывает один из N классов для всего изображения.
- Подготовка:
  - Соберите датасет со структурами изображение + `image.json` и меткой внутри (в `annotations[0].label` или другая логика).
  - В `Trainer` указывайте `model_type='classification'` и `num_classes` равный числу классов.
- Вызов (UI): во вкладке "Обучение" выберите модель классификации и нажмите "Запустить обучение"; выберите папку с датасетом.

6.2 Простой детектор (single-bbox)
- Поведение: модель предсказывает класс и один bbox на изображение (normalized x,y,w,h).
- Подготовка: используйте JSON аннотации с координатами и меткой.
- Ограничения: текущая реализация — упрощённая; работает только с одним боксом, который берётся из первой аннотации в `json`.
- Рекомендации: для корректной работы детектора указывайте `num_classes` в `Trainer` равным числу меток.

6.3 Рекомендации по гиперпараметрам
- epochs: 10..200 (в зависимости от датасета). Для тестов можно задать 1-5.
- batch_size: 8..128 (зависит от ОЗУ и GPU). На CPU используйте небольшие значения (8..32).
- learning rate: начальное значение 1e-3 (для `Adam`).
- оптимайзер: `Adam` — хорошая опция для старта.

7. Использование моделей: предсказание и экспорт
-----------------------------------------------
1) Откройте вкладку "Предсказание".
2) Нажмите "Загрузить фото для предсказания" и выберите картинку.
3) Убедитесь, что подходящая модель выбрана в верхнем выпадающем меню.
4) Нажмите "Сделать предсказание". Результат отобразится наложением bbox и текста с классом + confidence.
5) Чтобы экспортировать вариант с изображением и JSON, нажмите "Экспорт с метками" и укажите путь.

Формат JSON при экспорте (пример):
{
  "predictions": [
    {"class": 0, "confidence": 0.92, "bbox": [0.1, 0.15, 0.5, 0.45]}
  ]
}

9. Советы по отладке и анализ логов (common issues)
--------------------------------------------------
- "oneDNN custom operations are on..." — это просто INFO от TensorFlow. Он не говорит о проблемах. Если хотите скрыть такие INFO-сообщения:
  - В PowerShell: `$env:TF_CPP_MIN_LOG_LEVEL = 2; python -m vision_desktop_app.training.dataset_utils`
  - Или в коде (до импорта tensorflow): `import os; os.environ['TF_CPP_MIN_LOG_LEVEL'] = '2'`.
- "No module named 'PySide6'": активируйте виртуальное окружение и установите зависимости (`pip install -r requirements.txt`).
- "No module named 'vision_desktop_app.main'": приложение нужно запускать из корня проекта или через `run_app.py`; если запускаете файл напрямую, убедитесь что `sys.path` содержит путь топ-локации (корня проекта).
- "Received a label value X outside valid range" (Keras error): это означает, что значение метки больше, чем num_classes-1; проверьте, что `num_classes` соответствует вашим меткам. Для тестов используйте `num_classes = len(CLASS_NAMES)`.
- UI не появился/GUI не открывается: возможные причины —
  - Можете запускать приложение на headless-сервере (без GUI) — окно не появится.
  - PySide не установлен в активном venv.
  - Ошибка импорта или исключение во время инициализации UI — проверьте `stdout/stderr` в терминале для trace.

10. Как добавлять новые модели и конфигурации
-------------------------------------------
1) Добавьте запись в `vision_desktop_app/models/model_manager.py` в `DEFAULT_MODELS` (для быстрого добавления в UI).
2) Добавьте бэкенд модели в `vision_desktop_app/training/model_templates.py`: реализуйте функцию, возвращающую keras.Model.
3) Убедитесь, что `Trainer` поддерживает нужный тип (classification/detection) и передаёт `num_classes` в модель.
4) Чтобы UI отображал новые параметры модели, обновите `vision_desktop_app/ui/main_window.py` либо используйте конфиг в `models/<name>/config.json`.
