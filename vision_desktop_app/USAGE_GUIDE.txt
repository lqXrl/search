1) Краткое описание
-----------------
Проект: desktop-приложение (PySide6) на Python, использующее TensorFlow для обучения и предсказания.
Приложение поддерживает аннотацию изображений (bounding box), сохранение аннотаций (.json), обучение моделей (classification/detection), переключение моделей и простую визуализацию предсказаний.

2) Установка зависимостей
------------------------
Рекомендуется создать и использовать виртуальное окружение (Windows PowerShell):

```powershell
cd C:/Users/.../Desktop/search
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r vision_desktop_app/requirements.txt
```

Важно: запускать команды надо из корня проекта (`C:/Users/.../Desktop/search`) или из `vision_desktop_app` в зависимости от указаний дальше.

3) Быстрый старт (GUI)

Если вы используете `run_app.py`, то тоже можно:


4) Аннотация изображений
------------------------
1. Перейдите в вкладку "Аннотация" в приложении.
2. Загрузите изображение (кнопка "Загрузить изображение").
3. Введите название метки в поле (например, "person" или "box").
4. Нарисуйте bounding box мышью: нажмите левую кнопку для начала, отпустите — рамка будет сохранена.
5. Повторите для всех объектов. Нажмите "Сохранить аннотацию" — метки сохранятся в .json (рядом с изображением).

Формат .json:
{
  "image": "image_name.png",
  "annotations": [{"label":"person","x":10,"y":20,"w":50,"h":60}]
}

5) Обучение моделей
------------------
1. Переключитесь во вкладку "Обучение".
2. Выберите модель в выпадающем списке. По умолчанию доступны модели: inside_outside, station_objects, faces, ground_surface, space_objects.
3. Укажите количество эпох и batch size.
4. Нажмите "Запустить обучение" и выберите папку с данными (изображения + .json-аннотации).

Пример структуры данных:

- data/my_dataset/
  - image_0000.png
  - image_0000.json
  - image_0001.png
  - image_0001.json
  ...

Важно: если у вас несколько классов, убедитесь, что `num_classes` соответствует всем меткам.

Примечание: текущая реализация detection — простой регрессор, поддерживающий по одному bbox на изображение; для полноценного детектора потребуется расширение.

6) Предсказание и экспорт
------------------------
1. Перейдите во вкладку "Предсказание".
2. Загрузите изображение для предсказания.
3. Проверьте, что выбрана правильная модель (список моделей берет конфиги в `models/<model_name>/config.json`).
4. Нажмите "Сделать предсказание".
5. Для экспорта сохраните изображение с предсказанными bbox и сгенерированный JSON (показывает класс и confidence).

7) Генерация тестового датасета
------------------------------
В проекте есть генератор синтетических данных для быстрых тестов:

```powershell
python -m vision_desktop_app.training.sample_data_generator
```

Он создаст `data/sample_dataset` с изображениями и `.json` аннотациями.

9) Troubleshooting: частые сообщения и ошибки
---------------------------------------------
- "No module named 'PySide6'": проверьте виртуальное окружение и установку зависимостей (`pip install -r requirements.txt`).
- "No module named 'vision_desktop_app.main'": запускайте приложение из корня проекта `C:/Users/.../search` или из `C:/Users/В.../vision_desktop_app`/`python -m vision_desktop_app.main`. Альтернатива — запуск `run_app.py` в корне проекта.
- TensorFlow INFO: "oneDNN custom operations are on ..." — это информационное сообщение, не ошибка. Чтобы скрыть INFO:
  - В PowerShell: `$env:TF_CPP_MIN_LOG_LEVEL = 2; python -m vision_desktop_app.training.dataset_utils`
  - Или в коде перед `import tensorflow as tf`: `os.environ['TF_CPP_MIN_LOG_LEVEL'] = '2'`.
- Label value outside range during training: если Keras пишет `Received a label value 3 outside the valid range [0, 2)`, значит в обучающих данных есть метка > num_classes-1. Проверьте `num_classes` и соответствие меток. В примере `run_example.py` было исправлено, чтобы автоматически выставлять `num_classes = len(CLASS_NAMES)`.
